Practica 1

 Servidor NFS (192.168.0.199)
 Instalación de NFS
sudo dnf install nfs-utils -y

Instala las herramientas necesarias para configurar y utilizar el servicio NFS.

 Activar servicio NFS
sudo systemctl enable --now nfs-server

Habilita el servicio NFS para que inicie automáticamente y lo ejecuta inmediatamente.

 Crear directorio compartido
sudo mkdir /OS3

Crea el directorio que será compartido en la red.

 Crear archivos
cd /OS3
for i in {1..100}; do sudo touch Adrian${i}.txt; done

Genera 100 archivos dentro del directorio para pruebas.

 Asignar permisos
sudo chmod -R 755 /OS3

Otorga permisos de lectura y ejecución a todos los usuarios.

sudo chown -R nfsnobody:nfsnobody /OS3

Asigna el propietario del directorio al usuario utilizado por NFS.

 Configuración de Firewall
sudo systemctl enable --now firewalld
sudo firewall-cmd --permanent --add-service=nfs
sudo firewall-cmd --permanent --add-service=mountd
sudo firewall-cmd --permanent --add-service=rpc-bind
sudo firewall-cmd --reload

Permite el tráfico necesario para el funcionamiento de NFS.

 Desactivar SELinux (modo permisivo)
sudo setenforce 0

Desactiva temporalmente las restricciones de SELinux para evitar bloqueos en el acceso NFS.

 Configurar directorios compartidos
sudo nano /etc/exports

Agregar:

/OS3 192.168.0.198(rw,sync,no_subtree_check)

Define qué directorio se comparte, con qué cliente y con qué permisos.

 Aplicar configuración
sudo exportfs -rav

Actualiza y aplica la configuración de los recursos compartidos.

 Cliente NFS (192.168.0.198)
 Instalar cliente NFS
sudo dnf install nfs-utils -y

Instala las herramientas necesarias para conectarse a un servidor NFS.
Crear punto de montaje
sudo mkdir /mnt/nfs

Crea el directorio donde se montará el recurso compartido.

Montar recurso NFS
sudo mount 192.168.0.199:/OS3 /mnt/nfs

Monta manualmente el directorio compartido del servidor.

Verificar archivos
ls /mnt/nfs

Permite visualizar los archivos compartidos desde el servidor.

Montaje Automático
Editar fstab
sudo nano /etc/fstab

Agregar:

192.168.0.199:/OS3 /mnt/nfs nfs defaults 0 0

Configura el montaje automático al iniciar el sistema.

Probar configuración
sudo umount /mnt/nfs
sudo mount -a

Desmonta y vuelve a montar todos los sistemas definidos en fstab para validar.

Prueba Final
Reiniciar sistema
sudo reboot
Verificar montaje
ls /mnt/nfs

Confirma que el directorio se monta automáticamente y los archivos están disponibles.

Practica 2
 Instalación de SAMBA
sudo dnf install samba samba-client samba-common -y

Instala los paquetes necesarios para habilitar el servicio Samba en el sistema.
Incluye herramientas tanto para servidor como para cliente.

 Activación de Servicios
sudo systemctl enable --now smb nmb

smb: Servicio principal para compartir archivos

nmb: Servicio para resolución de nombres en red (NetBIOS)

Se habilitan para iniciar automáticamente y se ejecutan inmediatamente.

 Creación de Directorio Compartido
sudo mkdir /samba

Se crea la carpeta que será compartida en la red.

👥 Creación de Grupo y Usuario
sudo groupadd smbgrupo
sudo useradd -m -G smbgrupo usuario1
sudo passwd usuario1

Se crea un grupo para gestionar permisos

Se crea un usuario y se asigna al grupo

Se establece una contraseña para autenticación

 Registro del Usuario en SAMBA
sudo smbpasswd -a usuario1

Permite que el usuario pueda autenticarse específicamente en el servicio Samba.

 Asignación de Permisos
sudo chown -R usuario1:smbgrupo /samba
sudo chmod -R 777 /samba

Se asigna la propiedad del directorio al usuario y grupo

Se otorgan permisos completos (lectura, escritura y ejecución)

 Nota: En entornos productivos, se recomienda usar permisos más restrictivos.

 Configuración del Firewall
sudo systemctl enable --now firewalld
sudo firewall-cmd --permanent --add-service=samba
sudo firewall-cmd --reload

Permite el tráfico necesario para Samba a través del firewall del sistema.

 Configuración de SELinux
sudo setenforce 0

Desactiva SELinux temporalmente para evitar bloqueos en la configuración.

 En producción, se recomienda configurar políticas en lugar de desactivarlo.

 Configuración del Archivo smb.conf
sudo nano /etc/samba/smb.conf

Se agrega la siguiente configuración:

[samba]
 path = /samba
 browsable = yes
 writable = yes
 valid users = @smbgrupo
 guest ok = no
 create mask = 0777
 directory mask = 0777
Explicación:

path: Ruta del recurso compartido

browsable: Visible en la red

writable: Permite escritura

valid users: Restringe acceso al grupo

guest ok: Desactiva acceso anónimo

create mask / directory mask: Permisos por defecto

 Verificación de Configuración
testparm

Valida que la configuración de Samba no tenga errores.

Reinicio del Servicio
sudo systemctl restart smb nmb

Aplica todos los cambios realizados en la configuración.

Creación de Archivos de Prueba
cd /samba
for i in {1..100}; do touch adrian${i}.txt; done

Genera 100 archivos para validar el acceso desde clientes remotos.

Acceso desde Windows
\\192.168.0.199\samba

Permite acceder al recurso compartido desde un equipo Windows utilizando la red local.

Verificación de Sincronización
tail -f /samba/adrian99.txt

Permite monitorear cambios en tiempo real para comprobar la comunicación entre sistemas.

Práctica 3 –
Configuración de Samba como AD DC en Oracle Linux

IP del servidor: 192.168.0.199
Interfaz de red: eth0
Dominio: OS3.inet
Controlador: dc1
Usuario de prueba: lanegracubana

Paso 0: Preparar el sistema
# Cambiar el hostname del servidor
hostnamectl set-hostname dc1.inet
hostname
hostname -f

Explicación: Establece un nombre único para tu servidor. Esto es crítico para Active Directory.

# Acceder como root
sudo su
# Configurar hosts
echo "192.168.0.199 dc1.OS3.inet OS3.inet dc1" >> /etc/hosts

Explicación: Asocia IP y nombre de host para resolución interna de nombres.

# Actualizar el sistema
dnf -y update

# Instalar repositorios y herramientas adicionales
dnf install -y oracle-epel-release-el9
sudo dnf config-manager --set-enabled ol9_codeready_builder
dnf makecache
dnf -y install wget tar gcc make python3-devel

Explicación: Actualiza paquetes y prepara herramientas de compilación.

Paso 1: Descargar e instalar Samba
# Crear carpeta y descargar Samba
mkdir -p /samba && cd /samba
wget https://download.samba.org/pub/samba/stable/samba-4.16.2.tar.gz
tar -zxvf samba-4.16.2.tar.gz
cd samba-4.16.2/
# Instalar dependencias para compilar Samba
sudo dnf -y install docbook-style-xsl python3-markdown bison dbus-devel flex gdb \
gnutls-devel jansson-devel keyutils-libs-devel krb5-workstation \
libacl-devel libaio-devel libarchive-devel libattr-devel libblkid-devel \
libtasn1 libtasn1-tools libxml2-devel libxslt lmdb-devel openldap-devel \
pam-devel perl perl-ExtUtils-MakeMaker perl-Parse-Yapp popt-devel \
python3-cryptography python3-dns python3-gpg readline-devel rpcgen \
systemd-devel zlib-devel perl-JSON gpgme-devel screen

Explicación: Estas librerías son necesarias para soporte LDAP, Kerberos, seguridad y administración de Samba.

# Configurar la compilación de Samba
./configure --prefix=/usr/local/samba --enable-fhs --with-ads --with-systemd

# Compilar Samba
make -j$(nproc)
make install

# Actualizar PATH
echo 'export PATH=/usr/local/samba/bin:/usr/local/samba/sbin:$PATH' >> ~/.bash_profile
source ~/.bash_profile

Explicación: Esto instala Samba en /usr/local/samba y hace que los comandos de Samba sean globalmente accesibles.

Paso 2: Configurar librerías y SELinux
# Configurar ld.so para librerías Samba
echo "/usr/local/samba/lib" > /etc/ld.so.conf.d/samba.conf
echo "/usr/local/samba/lib64" >> /etc/ld.so.conf.d/samba.conf
ldconfig

# Desactivar temporalmente SELinux
setenforce 0

Explicación: Permite que las librerías de Samba se carguen correctamente y SELinux no bloquee el servicio mientras configuramos AD.

Paso 3: Provisionar el dominio
samba-tool domain provision \
--use-rfc2307 \
--interactive \
--option="interfaces= lo eth0" \
--option="bind interfaces only=yes"

Realm: OS3.INET

Domain: OS3

Server Role: dc

DNS backend: SAMBA_INTERNAL

DNS forwarder: 8.8.8.8

Contraseña de administrador: Teletubies123

Explicación: Crea el dominio Active Directory con soporte DNS interno, mapeando interfaces de red para la comunicación interna.

Paso 4: Configurar Kerberos y PATH
# Copiar krb5.conf generado por Samba
cp /usr/local/samba/var/lib/samba/private/krb5.conf /etc/krb5.conf

Explicación: Esto permite que Kerberos use la configuración de Samba para autenticación.

Paso 5: Configurar firewall
firewall-cmd --permanent --add-service={ldap,ldaps,kerberos,dns}
firewall-cmd --permanent --add-port={53,88,135,139,389,445,464,636,3268,3269}/tcp
firewall-cmd --permanent --add-port={53,88,123,138,389,464}/udp
firewall-cmd --permanent --add-port=49152-65535/tcp
firewall-cmd --reload

Explicación: Permite que los servicios esenciales de AD (LDAP, DNS, Kerberos, SMB) funcionen correctamente en la red.

Paso 6: Configurar DNS en la interfaz
nmcli con mod eth0 ipv4.dns 192.168.0.199
nmcli con mod eth0 ipv4.ignore-auto-dns yes
nmcli con up eth0

Explicación: Hace que tu servidor use su propia IP para resolución DNS del dominio.

Paso 7: Crear usuario administrador del dominio
samba-tool domain passwordsettings set --complexity=off

samba-tool user create lanegracubana "20252263" \
--given-name="La Negra" --surname="Cubana" \
--mail=lanegracubana@os3.inet

samba-tool group addmembers "Domain Admins" lanegracubana
samba-tool group listmembers "Domain Admins"

Explicación:

Desactiva la complejidad de contraseñas (para pruebas)

Crea un usuario con privilegios de administrador en AD


nano /etc/systemd/system/samba.service

[Unit]
Description=Samba AD Daemon
After=network.target

[Service]
Type=forking
ExecStart=/usr/local/samba/sbin/samba
ExecStop=/bin/kill -TERM $MAINPID

[Install]
WantedBy=multi-user.target

systemctl daemon-reexec
systemctl daemon-reload
systemctl enable samba
systemctl start samba


Paso 8: Verificar dominio
samba-tool domain info 192.168.0.199
samba-tool user list
samba-tool computer list

Explicación: Comprueba que tu AD está activo y muestra usuarios y computadoras registradas.

Paso 9: Agregar registros DNS
samba-tool dns add 192.168.0.199 OS3.inet www A 192.168.0.100 -U lanegracubana
